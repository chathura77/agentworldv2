Public VM Security Review

No reportable application-code security vulnerabilities were found in the current static frontend scope. The largest practical risk is operational: accidentally exposing development tooling or a writable/root container.

Required Production Posture

  • Serve only dist/ from an unprivileged static server.
  • Do not expose Vite, Node, npm, Docker API, or package-manager ports.
  • Publish only 80/443 externally and terminate TLS.
  • Use the included Nginx security headers.
  • Keep SSH restricted to trusted admin IPs.
  • Run npm run check and scan the final image before deploy.

Evidence

  • npm test: 47 tests passed.
  • npm run lint: passed.
  • npm run typecheck: passed.
  • npm audit --audit-level=moderate: zero vulnerabilities.
  • npm run build: passed.
  • Docker build runs the same combined check gate.

See docs/security-review.md and docs/production-readiness.md for the full checklist.
